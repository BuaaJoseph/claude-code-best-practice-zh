# 验证检查清单 — Subagents 报告

规则会随着时间累积。每次 workflow-changelog 运行必须执行指定深度的所有规则。当发现现有规则应该捕获但没有捕获或捕获不够深的新类型漂移时，在此追加新规则。

## 深度级别

| 深度 | 含义 | 示例 |
|-------|---------|---------|
| `exists` | 检查章节/表/文件是否存在 | "报告中是否有 Memory Scopes 表？" |
| `presence-check` | 检查特定项目是否存在 | "Frontmatter Fields 表中是否有 `color` 字段？" |
| `content-match` | 与源逐字比较实际值 | "`model` 字段描述是否与官方文档匹配？" |
| `field-level` | 验证每个单独字段都已记录 | "官方文档中的每个 frontmatter 字段是否出现在表中？" |
| `cross-file` | 同一值必须在多个文件中匹配 | "CLAUDE.md agent 部分是否与报告的字段列表匹配？" |
