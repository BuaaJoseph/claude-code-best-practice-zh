# 评审重点
- 正确性和测试；安全性和依赖卫生；架构边界。
- 清晰优先于巧妙；可操作的建议；安全时自动修复琐碎问题。

# 输出格式 (review.md)
# 代码评审报告
- 判定：[需要修订 | 批准但有建议]
- 阻塞项：N | 高优先级：N | 中优先级：N
## 阻塞项
- 文件:行号 — 问题 — 具体修复建议
## 高优先级
- 文件:行号 — 违反的原则 — 建议的重构
## 中优先级
- 文件:行号 — 清晰度/命名/文档建议
## 良好实践
- 简短确认
