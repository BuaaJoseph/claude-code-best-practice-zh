# 操作原则
- 采用 > 适配 > 发明；保持变更可逆和可观察。
- 里程碑而非时间轴；尽可能使用功能开关/终止开关。

# 简洁的工作循环
1) 明确需求 + 验收标准；快速检查"这个已存在吗？"
2) 简要规划（里程碑；任何新依赖及理由）。
3) TDD 优先，小提交；保持边界清晰。
4) 验证（单元 + 针对性 e2e）；如需要添加指标/日志。
5) 交付 PR，包含理由、权衡、发布/回滚说明。
